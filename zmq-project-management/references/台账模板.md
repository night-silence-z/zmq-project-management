# 台账模板（用于生成 `00_管理/台账.md`）

> 建档时按"模板"区创建文件，占位符 `{}` 替换为实际内容；"示例"区仅供参照格式与颗粒度，创建时不要带入。

## 模板

```markdown
# {项目名} · 台账

## 一、待办

| # | 事项 | 状态 | 备注 |
| -- | -- | -- | -- |
| 1 | {事项} | 进行中/待启动/已完成 | {备注} |

## 二、决策与待确认

### 已决策

| 编号 | 日期 | 决策 | 原因 | 影响范围 |
| -- | -- | -- | -- | -- |
| D-001 | {YYYY-MM-DD} | {决策内容} | {为什么} | {影响哪些产出/规则} |

### 待确认

| 编号 | 事项 | 决策人 | 截止条件 | 默认方案 | 影响范围 |
| -- | -- | -- | -- | -- | -- |
| P-001 | {待确认事项} | {谁拍板} | {到什么时候/什么事件前必须定} | {不确认时按什么执行} | {影响什么} |

## 三、迭代记录（只增不删）

| 时间 | 阶段 | 事件类型 | 内容 | 原因 | 影响与关联文档 |
| -- | -- | -- | -- | -- | -- |

> 一致性扫描水位：已扫至第 0 条（{YYYY-MM-DD}）
```

## 示例（参照颗粒度，注意"推翻"行的写法）

| 时间 | 阶段 | 事件类型 | 内容 | 原因 | 影响与关联文档 |
| -- | -- | -- | -- | -- | -- |
| 2026-07-17 10:20 | M1 | 进展 | 落地页原型 v3 完成（claude） | — | 01_产出/原型/当前/落地页-v3.html |
| 2026-07-17 11:05 | M1 | 决策 | D-002：留存入口统一命名"订阅" | 多入口语义不一致会造成状态混乱 | 原型 v3、PRD v2 待同步 |
| 2026-07-17 15:40 | M1 | 推翻 | 原定首页轮播 → 改为静态首图 | 线上数据显示轮播点击率不足 1% | 原型升 v4；PRD §3.2 同步修改 |
| 2026-07-17 16:12 | M1 | 纠错 | 楼盘卡字段原按 v0.2 契约 → 改按 v0.4 实时聚合 | 后端接口文档更新 | 契约清单 v0.4；已全链路扫描代码/文档/测试 |

要点：推翻和纠错必须写清"原来是什么 → 改成什么 → 为什么"；多 AI 协作时在"内容"列注明执行者；❌"修了一些问题、优化了体验"这种写法不可追溯，禁止。
