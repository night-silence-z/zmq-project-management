# 启动卡模板（立项访谈收敛后，写入 `00_管理/README.md` 的"启动卡"节）

> 一页以内；用户签核后启动卡冻结，后续变更走台账决策记录，不直接改卡。

## 模板

```markdown
## 启动卡

**一句话**：本项目交付 {X}，给 {Y} 用来 {Z}，验收标准是 {W}。

| 三要素 | 内容 |
| -- | -- |
| 最终产出物 | {形态：如 单文件 HTML 报告 / 小程序页面 / PRD+原型} |
| 受众与用途 | {谁看/谁用，拿去做什么决策或动作} |
| 验收标准 | {可观察、可核验的标准，不写"效果好"} |

| 范围 | 内容 |
| -- | -- |
| 本期做 | {…} |
| 明确不做 | {…} |
| 顺延 | {…（顺延到何时/什么条件触发）} |

- 关键约束：{口径/技术/合规/资源约束}
- 待确认：{P-00N，联动台账待确认表；无则写"无"}
- 签核：{用户} 于 {YYYY-MM-DD} 确认
```

## 示例

**一句话**：本项目交付一份冷启动留存分析报告（单文件 HTML），给业务负责人用来决定下季度产品与运营的 P0 动作，验收标准是结论有分层数据支撑、动作拆到执行者并带验证指标。

| 范围 | 内容 |
| -- | -- |
| 本期做 | 6 月数据的留存分层分析＋分群运营建议 |
| 明确不做 | 线索成本分析、投放预算测算 |
| 顺延 | 日常监控看板（数据口径统一后启动） |
